# 初階範例
## 初體驗

### 📚 工作流程說明

這個 n8n 工作流程是專為初學者設計的第一個範例。它展示了最基礎的資料處理流程：從模擬的客戶資料庫中獲取資料，透過 Set 節點整理與篩選所需欄位，最後將整理好的資訊透過模擬的訊息服務發送出去。這能幫助你快速理解 n8n 的資料流向與基本節點操作。

#### 📋 節點詳細說明

1. **📝 Sticky Note（便利貼）**
   - **功能**：流程中的註解說明。
   - **內容**：標示此為「初體驗」範例，並說明各區塊的功能（如取得數據、提取資料）。

2. **👆 Manual Trigger（執行工作流）**
   - **功能**：手動點擊「Execute Workflow」按鈕來啟動流程。
   - **用途**：適合開發測試或非定期執行的任務。

3. **🗃️ Customer Datastore (n8n training)**
   - **功能**：模擬從外部資料庫讀取客戶資料。
   - **操作**：使用 `getAllPeople` 取得所有模擬的客戶名單。
   - **回傳資料**：包含 ID、姓名、Email、備註、國家等欄位。

4. **🔄 Edit Fields (Set)**
   - **功能**：資料清洗與欄位對應。
   - **欄位對應**：
     - `id` ← `$json.id`
     - `name` ← `$json.name`
     - `email` ← `$json.email`
     - `notes` ← `$json.notes`
     - `country` ← `$json.country`
   - **目的**：確保傳遞給下一個節點的資料格式是乾淨且正確的。

5. **💬 Customer Messenger (n8n training)**
   - **功能**：模擬發送訊息通知。
   - **內容**：將整理好的客戶資訊（ID, Name, Email 等）組合成訊息發送。
   - **輸出範例**：
     ```
     Customer Information:
     ID: 1
     Name: John Doe
     Email: john@example.com
     ...
     ```

#### 🎯 學習重點

- **手動觸發**：了解如何手動啟動 Workflow。
- **資料獲取**：體驗從資料來源（Datastore）撈取資料。
- **資料轉換**：學習使用 Set 節點（Edit Fields）來重新命名或篩選資料欄位。
- **資料使用**：在最終節點（Messenger）中使用表達式（Expression）引用前序節點的資料。

#### 💡 實際應用場景

- 定期從 CRM 系統撈取新客戶名單並通知業務團隊。
- 將資料庫中的訂單資料整理後發送至 Slack 或 Email。
- 簡單的資料備份或轉移任務。

#### ⚙️ 設定步驟

1. **開啟流程**：匯入此範例工作流程。
2. **執行測試**：點擊畫面下方的「Execute Workflow」按鈕。
3. **檢視結果**：觀察各節點的執行狀態（綠色勾勾），並點擊 Customer Messenger 節點查看輸出的訊息內容。
