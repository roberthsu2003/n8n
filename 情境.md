## 💻 n8n 與 Google 應用整合的辦公室範例

### 1. Google Sheets (試算表) 為核心的數據處理自動化

Google Sheets 在許多辦公室中是核心數據庫。n8n 可以將其變成一個自動化的數據中心。

| 流程情境 | 整合方式 | 核心價值 |
| :--- | :--- | :--- |
| **自動化報告生成** | **觸發：** 定時器（每天早上 9 點）。**動作：** 從 CRM 或廣告平台（如 Google Ads, Facebook Ads）獲取數據，**寫入** Google Sheets 中的不同工作表，然後將彙整好的數據通過 Email **發送**給團隊主管。 | **準時**、**零誤差**的每日或每週報告，取代人工手動彙整。 |
| **表單數據處理** | **觸發：** Google Forms 有新提交或有人**新增** Google Sheets 列。**動作：** 根據表單內容（例如：產品訂購、請假申請）自動**建立** Trello 任務、**發送**確認 Email，並**更新**另一個總表。 | **即時響應**表單提交，確保數據被正確導入後續流程。 |

### 2. Google Drive (雲端硬碟) 的文件管理自動化

管理大量的電子文件夾和文件權限可能非常耗時。

| 流程情境 | 整合方式 | 核心價值 |
| :--- | :--- | :--- |
| **合約文件分類與通知** | **觸發：** 有新文件**上傳**到 Google Drive 中的特定「待處理合約」資料夾。**動作：** 1. 檢查檔案名稱是否符合命名規範。 2. 自動將檔案**移動**到對應的客戶/專案資料夾。 3. 在 Slack 或 Email **通知**法務部或業務主管進行審批。 | **標準化**文件存檔流程，避免文件混亂和遺失。 |
| **定期備份與清理** | **觸發：** 定時器（每月 1 號）。**動作：** 搜尋 Drive 中超過 3 年沒有被修改的檔案，將其**移動**到「Archive」資料夾，或發送通知讓使用者確認是否刪除。 | **優化**雲端儲存空間使用，保持工作區整潔。 |

### 3. Gmail 與 Google Calendar (日曆) 的溝通排程自動化

將 Email 內容自動轉換為任務，並同步會議排程。

| 流程情境 | 整合方式 | 核心價值 |
| :--- | :--- | :--- |
| **Email 轉換為任務** | **觸發：** 收到特定 Email 地址或主旨的信件。**動作：** 1. **解析** Email 內容。 2. 自動在專案管理工具 (如 ClickUp, Asana) **建立**任務。 3. **標記**該 Email 為已讀或移動到特定資料夾。 | 確保重要的客戶請求或支援工單**不會被遺漏**，並自動進入工作流程。 |
| **會議排程自動化** | **觸發：** 在 CRM 系統中將交易狀態標記為「安排會議」。**動作：** 1. 自動在 Google Calendar 上**建立**一場會議。 2. 邀請內部團隊成員。 3. 發送附帶視訊會議連結（Meet）的邀請給客戶。 | **加速**業務排程，減少反覆確認時間。 |

## GitHub Repo整合

太棒了！將 **GitHub 儲存庫 (Repository, 簡稱 Repo)** 數據整合到 n8n 中是**完全可行**的，而且對於開發者、專案經理或上課展示「DevOps 自動化」來說，GitHub 是一個非常好的數據來源。

n8n 有一個強大的 **GitHub 節點**，可以直接與 GitHub 的 API 進行互動。

## 💻 GitHub 數據整合的 n8n 應用範例

您可以使用 n8n 從您的個人或組織的儲存庫中擷取以下數據：

### 1. 專案狀態追蹤與通知

| 數據類別 | n8n 應用情境 | 整合工具範例 |
| :--- | :--- | :--- |
| **Pull Requests (PRs)** | **觸發：** 儲存庫中有新的 PR 被建立或更新。**動作：** 1. 自動檢查 PR 標題是否符合規範（例如：是否包含 `[FIX]` 或 `[FEAT]`）。 2. 發送 Slack 訊息到開發頻道，提醒相關人員進行審核 (Review)。 3. 如果 PR 已經合併，自動在 Trello/Jira 中關閉相關的 Issue。 | GitHub 節點, Slack, Trello / Jira |
| **Issues (問題追蹤)** | **觸發：** 有新的 Issue 被提交。**動作：** 1. 自動為 Issue 加上標籤 (Label)，例如：根據標題關鍵詞自動標註 `bug` 或 `enhancement`。 2. 如果是標記為 `urgent` 的 Issue，發送 Email 或 Line 通知給部門主管。 | GitHub 節點, Line, Gmail / Outlook |

### 2. 儲存庫數據分析與報告

| 數據類別 | n8n 應用情境 | 整合工具範例 |
| :--- | :--- | :--- |
| **Commit 歷史記錄** | **觸發：** 定時器（每週一早上）。**動作：** 1. 擷取過去一週的 Commit 數量和貢獻者 (Contributor) 列表。 2. 將數據寫入 Google Sheets，生成一個「每週程式碼活動報告」。 | GitHub 節點 (List Commits), Google Sheets |
| **Star/Fork 數量追蹤** | **觸發：** 定時器（每日）。**動作：** 1. 獲取您的公開儲存庫的最新 Star 和 Fork 數量。 2. 將數據存入數據庫 (如 MySQL/Postgres)，用於長期趨勢分析。 3. 當 Star 數量達到特定里程碑時 (例如：突破 1000)，自動發送慶祝訊息到 Discord 或 Slack。 | GitHub 節點 (Get Repository), MySQL, Discord |

### 3. 自動化維護工作

| 數據類別 | n8n 應用情境 | 整合工具範例 |
| :--- | :--- | :--- |
| **文件自動更新** | **觸發：** 專案版本號在 `package.json` 中被更新。**動作：** 1. n8n 讀取新的版本號。 2. 自動修改 `README.md` 或其他文件中的版本資訊，並**創建一個 Commit** 將其推送到儲存庫。 | GitHub 節點 (Create File, Edit File) |

## 🔑 n8n 整合 GitHub 的關鍵

要開始使用，您需要在 n8n 中設定 **GitHub 認證 (Credentials)**：

1.  **個人存取權杖 (Personal Access Token, PAT)：** 這是最常用的方法。您需要在 GitHub 帳號的 **Settings -> Developer settings -> Personal access tokens** 中生成一個 PAT。
2.  **權限範圍 (Scopes)：** 在生成 PAT 時，您必須選擇正確的權限範圍（例如：如果您需要讀取 Issue，則需要 `repo` 相關的權限）。

在您的課程中，展示如何設定 PAT 以及如何用 **GitHub 節點**的 `List Issues` 或 `List Pull Requests` 操作來獲取結構化的 JSON 數據，將是非常有價值的教學內容！
